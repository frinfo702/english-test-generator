# Tooling

## Build System

### Vite

The project uses **Vite 7** as the build tool and dev server.

- Config: `vite.config.ts`
- Dev server runs on `http://localhost:5173` with HMR.
- Production build outputs to `dist/`.
- The Vite config proxies `/api/transcribe` to the Wrangler dev server.

### TypeScript

**TypeScript ~5.9.3** with `strict` mode enabled.

- Config: `tsconfig.json` (references `tsconfig.app.json` and `tsconfig.node.json`)
- App source config: `tsconfig.app.json` — targets ES2022, uses `react-jsx` transform, includes `src/`
- Node config: `tsconfig.node.json` — for Vite and Wrangler config files
- `verbatimModuleSyntax` — type-only imports must use the `type` keyword.
- `noUnusedLocals` / `noUnusedParameters` — enforced at build time.

Run type-checking:

```bash
npx tsc -b
```

## Linters

### ESLint

**ESLint 9** with flat config format.

- Config: `eslint.config.js`
- Plugins:
  - `@eslint/js` — recommended rules
  - `typescript-eslint` — TypeScript-specific rules (strict)
  - `eslint-plugin-react-hooks` — React Hooks rules
  - `eslint-plugin-react-refresh` — HMR-compatible exports
- Ignores `dist/` directory.

Run:

```bash
npm run lint
```

### Prettier

Code formatting is handled by **Prettier** (configured in `package.json` or implicit defaults).

Run:

```bash
npm run format
```

There is no separate `.prettierrc` file — the default Prettier config is used, applied by the `format` script.

## Code Generators

### Audio Generation

`scripts/generate-audio.ts` is a TTS audio generator that:

1. Walks all question JSON files in `public/questions/`.
2. For files with `audioSegments` (dialogues) or `sentences` (narration), calls the **xAI TTS API** (`api.x.ai/v1/tts`).
3. Saves MP3 files to `public/audio/<task-dir>/<question-id>/`.
4. Uses a local cache (`public/audio-cache/`) to avoid re-fetching identical text+voice combinations.
5. Assigns voices deterministically based on a hash of the question and role — roles get unique voices from a pool of five (`ara`, `eve`, `leo`, `rex`, `sal`).

Usage:

```bash
# Requires XAI_API_KEY in .env.local
npm run generate-audio
```

### Question Generators

Question JSON files are generated by AI agents (Claude, etc.) using skill definitions stored in the `skills/` directory. These skills provide schema definitions and prompt templates for each task type (TOEFL Reading, TOEFL Writing, TOEIC Parts 5/6/7, etc.).

## CI Tooling

### GitHub Actions

Two workflows in `.github/workflows/`:

**`test.yml`** — runs on every push and pull request:

```yaml
- uses: actions/checkout@v4
- uses: actions/setup-node@v4
  with:
    node-version: 22
    cache: npm
- run: npm ci
- run: npm test
```

**`deploy.yml`** — runs on pushes to `main`:

```yaml
- run: npm ci
- run: npm run build
- run: npx wrangler pages deploy dist --project-name english-test-generator
  env:
    CLOUDFLARE_API_TOKEN: ${{ secrets.CF_API_TOKEN }}
```

### Cloudflare Wrangler

**Wrangler 4** handles Cloudflare Pages deployment and local Functions development.

- Config: `wrangler.jsonc`
- The `AI` binding enables Workers AI for speech transcription (Whisper).
- `pages_build_output_dir` is set to `dist`.
- `observability` is enabled with 100% head sampling for function logs.
